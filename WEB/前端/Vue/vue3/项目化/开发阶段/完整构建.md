组件测试

测试框架：vitest、Peeky

你的 Vue 应用中大部分内容都应该由组件测试来覆盖，我们建议每个 Vue 组件都应有自己的组件测试文件。

vitest：
1. 在每个项目目录下，执行：npm install -D vitest happy-dom @testing-library/vue
2. 修改vite.config.js，配置别名：
```js
// vite.config.js
import { defineConfig } from 'vite'

export default defineConfig({
  // ...
  test: {
    // 启用类似 jest 的全局测试 API
    globals: true,
    // 使用 happy-dom 模拟 DOM
    // 这需要你安装 happy-dom 作为对等依赖（peer dependency）
    environment: 'happy-dom'
  }
})
```
3. 修改package.json，配置运行指令
```json
{
  // ...
  "scripts": {
    "test": "vitest"
  }
}
```
4. 在顶层目录下新建test目录，所有测试文件都放在这里
5. 文件格式以`*.test.js`为后缀
6. npm test；在vscode可通过NPM SCRIPT栏，手动点击
7. **你修改哪个测试文件，test任务就会自动执行当前测试**，没找到指定测试文件的方式
